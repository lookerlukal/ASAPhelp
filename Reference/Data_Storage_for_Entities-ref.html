Data Storage for Entities

SURFACE/FUNCTION , EDGE/CURVE and LENS data currently share the same array for internal storage. In fact, they have equivalent storage locations.

Tips
  • As SURFACEs/FUNCTIONs, EDGEs/CURVEs and LENSES are defined, they fill the array at the next available location. This concept is important because SURFACEs/FUNCTIONs, EDGEs/CURVEs, and LENSES are referenced by OBJECTS, and objects are what ASAP ray traces. Objects use this database to model the physical geometry of the optical element.

  • Array Storage Order


    Surface 1optical
    Edge 2ellipse
    Edge 3ellipse
    Lens 4mirror
    Surface 5tube
    Surface 6torus
    Lens 7mangin
    . . .
  • Although individual entities fill the array at the next available database location, the entities take up only the memory needed to store the data of the entity at that location. Therefore, entities in the database locations do not necessarily take up the same amount of memory.
  • To reduce memory conflict, use the UPDATE command before updating any database locations. Entity database locations may be overwritten with a new entity. Normally, when a defined entity is updated at a later time, the new data overwrites the old. There is no memory conflict if the amount of new data of the entity is less than or equal to the old entity when updating the data base location. If the amount of new data is more than the old entity, the data of the updated entity corrupts the entity data in the next database location.
