DEFORM with Arrayed Sampled Surface

The example, Deforming to an Arrayed Sampled Surface, demonstrates not only the DEFORM command, but also several other commands and useful techniques for modeling complicated objects.

This example shows a base object that has perturbations on top of it in the form of a bumpy structure. The bumpy structure is part planar with a hemisphere on top of the plane and a smaller shell on top of the hemisphere. The entire structure may be considered as a unit cell. The unit cell is regularly repeated over the base object.

The basic technique is to model the unit cell as a single entity, replicate the unit cell with the ARRAY entity modifier, and then deform this to the base object. The key is the MAP command.

The MAP command produces a depth map, in the form of a distribution data file, of the current objects using the last WINDOW and PIXEL settings. The MAP command not only maps out the depths of the objects, but also two orthogonal slopes at each scan point. The MAP command syntax is: 

MAP [ name ] [ DEPTH [ d' [ d" ] ] ] [ SLOPES ]

where name is the name of the distribution data file, d' and d" are the depth coordinates, and SLOPES is the optional specification for calculating slopes.

Tips
  • The MAP command is different from FMAP in that the MAP command maps objects where FMAP maps surfaces. Now that a distribution data file describing the unit cell exists, you may use the SAMPLED command to model it as a surface.
  • When modeling your particular optical system, remember that objects created in this manner are in a sense discrete because of the method of construction (the depth and slopes of SAMPLED are linearly interpolated between points).
