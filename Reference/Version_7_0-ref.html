Release Notes for ASAP 7.0 Kernel as of January 19, 2001

Important Changes (no backward compatibility)

LENSES no longer treat zero curvature as special case of 4th-order aspheric.

PARTICLES with $FCN size distribution has new entry for integration samples.

FTSIZE defaults to 512 samples at startup (and after a RESET or END).

EMITTING DATA uses DUMP file wavelength only if current WAVELENGTH is zero.

New Commands

COLLECTION efficiency as a function of aperture size and acceptance angle.

LENS

ABERRATIONS of a centered set of conicoids.

VARIABLES for locally or globally optimizing a basic imaging system.

MINIMIZE weighted average or maximum RMS spot size subject to constraints.

STORE current design in an ASAP file and/or a CodeV sequence file.

DOME forms a single refractive element with at least one side a hemisphere.

DISPLAY

APPEND current data to the end of an existing distribution file.

EDGE

EXTEND linearly one or both ends of a curve.

VOXELS for tracking and visualizing volume fluence, absorption or irradiance.

MODEL

VOLUME scatter version of PARTICLE for use with new MEDIA SCATTER option.

SURFACE

CONDUIT sweeps a circle along a explicit cubic planar curve

Enhanced Commands
 
SURFACE
	REVOLUTION
		Can optionally put input data points into current 3D VECTOR file.
		Smart FIT option automatically finds optimum degree surface.
	USERFUNC/USERSAG/SAMPLED now handle ROTATEd versions better.
	Different SAMPLED surfaces can now access same distribution file.
	Cartesian (not polar) parameterization on holeless RECTangular apertures.
	ELLIPSOID can optionally be only a half instead of a full one.
OBJECT
	DEFORM short form
		Now exact when ACCURACY HIGH and the base is a SURFACE entity.
		Can use a general USERSAG definition as the deformation.
FFAD now exactly autoscales RMS spot-size ellipses.
MEDIA
	Can now be directly created from any of the glass catalogs.
	Optional SCATTER for direct inhomogeneous Monte-Carlo volume scattering.
LENS
	Basic surface extended to a conicoid minus a paraboloid (e.g. an aspheric).
	SEQUENCE can optionally take DIAMeters instead of HEIGHTS (semi-diameters).
	MANGIN can now be an APLANAT (to third-order), just like SINGLET.
RADIANT
	Automatically averages to compensate for smaller solid angles near poles.
	Optional LOW or HIGH resolution 3D shaded patterns in place of wireframes.
DISPLAY can pick out a specific 2D slice of a 3D distribution data file.
	MESH LOW or HIGH resolution 3D shaded display in place of simple wireframes.
	GRAPH uses new (hopefully improved) algorithm for axes numbers/tic-marks.
ACCURACY LOW/MED for some Bezier intersections improved without speed loss.
XMEMORY option to allocate ray storage entirely in RAM for optimum speed.
SCATTER
	TOWARDS can now treat more precisely an odd-shaped importance-area EDGE.
ALTER now has an option for every "n"th EDGE point or LENS conicoid.
CURVE
	SPLINE 3D smoother and can operate with some or all tangent vectors missing.
	COMPOSITE has new option for merging GAP points.
...CLIP option can now reference a (possibly ARRAYed) SURFACE entity.
HALT settings can now be applied on an object-by-object basis.
GAUSSIAN astigmatic mode can now be created from just one ray/beam.
MODEL
	...PLOT puts hemispherical output for all default angles into one DIS file.
FIELD can take an existing file and propagate it through a micro-structure.
VIOLATION now also controls stability errors for above FD-BPM algorithm.
TRACE STATS prints out range of indices in any inhomogeneous (GRIN) media.
	...PIXELS option can now override aspect ratio (vertical/horizontal widths).
SHAPE of a beam can be the fundamental mode of a symmetric step-index SLAB.
Macro Language

$CLEAR clears the output buffer for future $GRAB and/or $PLAY commands.

$COPY copies or appends an arbitrary existing file to one of a different name.

$SCR can write up to 344 characters per line to the current output device.

$VIEW can optionally send a file to a new viewer or add it to the current one.

$DISP immediately or automatically displays binary distribution files.

$PATH now closes default files in old directory and then reopens them in new.

Other Changes

ASAP-Basic can now handle more complex systems (for example, imported from CAD):

Several speed improvements making this the fastest version of ASAP yet.
