Syntax Rules for Tolerancing in the Editor

This topic describes syntax rules for tolerancing analysis in the Editor.


  1. A tolerance parameter must be numeric. Numeric entries must be in integer, floating point or exponential format. The following ASAP command scripts are valid: 
    OPTICAL Z <1> 0
OPTICAL X 1 <2.5>
OPTICAL Z 0 0 <-1E-3>
  2. A tolerance parameter can contain one of the following operators: @, +, -, *, or /. The following ASAP command script is valid: 
    OPTICAL Z <2*1.4> 1
  3. A variable referencing a numeric value may be used to define a tolerance parameter as shown in the ASAP command script below: 
    LENS_BK7=<1.51118331>
MEDIA;     (LENS_BK7)   'PMMA'
  4. A tolerance parameter cannot contain equations, axis, matrix values, built-in functions or macros. For example, the following ASAP command scripts are invalid: 
    OPTICAL Z <LOC> 1, where LOC is variable previously defined
OPTICAL Z <LOC=2.5*3> 1
POINTS Z 0   <1 1 1
                      0 2 1
                      1 3 1
                      3 3 1
                      4 2 1
                      3 1 1>
PLANE <Z>  0 1
  5. The MEDIA command must have one of the following forms: 
    MEDIA; <1.613> 'SK4' 
MEDIA 
   <1.00> 'GLASS'
   <1.30> 'MGFL2'

MEDIA;
   <1.51> 'GLASS'; <1.38> 'MGFL2'

MEDIA <1.5> ‘GLASS’ ; <1.38> ‘MGFL2’
  6. The following commands can contain tolerancing values:

    SINGLET, PLANE, BICONIC, ELLIPSOID, MEDIA, OPTICAL, TORUS, COATINGS/COATING, POINTS, SPHERICAL, PARABOLIC, TUBE, RECTANGLE, ELLIPSE, MIRROR, ROTATE, SHIFT, SCALE, MOVE, SWEEP, SOURCE, VARIABLE, GROUP.

  7. An ASAP command script line containing one of these commands (SURFACE, LENSES, EDGES, ENT OBJECT and ENT OBJ) must be followed by another ASAP command script line containing a command that is allowable for tolerancing (listed above): For example, 
    SURFACE
	PLANE Y <0>
  8. Define Figure of Merit (FOM). Note that all FOMs must be placed at the beginning of the *.inx file using the format: FOM ‘GrabString’ (similar to the $GRAB macro). “FOM” cannot be used as the name of a FOM function (that is, ‘GrabString’ cannot have the name, “FOM”).
