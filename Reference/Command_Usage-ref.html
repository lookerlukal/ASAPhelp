Command Usage

ASAP commands consist of up to 10,000 unformatted alphanumeric entries separated by one or more of blanks " " or commas ",".

Arbitrary string entries can be delimited by double quotes. Otherwise, literal entries consist of an unbroken string of alphanumeric characters (A through Z through 9, and _). They must start with a letter or underscore "_" and can be of any length (but only the first 8 may be significant).

Numeric entries can be in integer, floating point or exponential format. Exponential entries, for example "1.34E-5", cannot be longer than 40 characters.

Usually a command begins with a literal, and there is only one command in each input record. However, more than one command can be placed in a record by separating them with semicolons ";". Also a command can extend over more than one record. If the last non-blank character in an input record is a comma (,), the command continues to the next record.

Note to European ASAP Users

Decimal numbers must be entered with a U.S. decimal point, using the period (.) key. If you enter decimal numbers with the European decimal point, using the comma (,) key, ASAP treats it as a space, and your results will not be accurate.
