Reference (O-R)

Reference topics in this section begin with the letters "O" to "R".

  • OBJECT Command Overview
    In ASAP, an object is a bounded physical entity with optical properties. The OBJECT command associates information from the optical property databases with the system geometry databases (MEDIA, COATINGS, MODELS, SURFACES, EDGES, LENSES, and others). Objects are subsequently constructed from these databases.
  • Optical Properties
    Using ASAP, you can build optical system models by first creating optical property and system geometry databases. The databases have additional commands for direct data entry and commands called modifiers that allow you to alter the databases. As you build an object, you associate data from these databases to the object for ray tracing.
  • Optimizing a Centered Imaging System
    A lens can be optimized using a brute-force algorithm that finds either the nearest minimum or (given enough time) the global minimum, without requiring the merit function to be continuous (that is, no derivatives are used).

  • Polarization Ray Tracing Overview
    FRESNEL BOTH initiates polarization calculations by instructing ASAP to calculate the appropriate reflection and transmission coefficients at an OBJECT INTERFACE for each ray as a function of polarization and angle of incidence.
  • Polarization Effects of a Sapphire Dome
    A birefringent material can change the polarization apodization and therefore the irradiance across the beam. This change is sometimes referred to as a polarization aberration. In the example of this topic, see how the ordinary and extraordinary indices are defined.
  • Polychromatic Diffraction MTF of a Doublet System
    The calculation of the polychromatic Modulation Transfer Function (MTF) involves performing an Fast Fourier Transform (FFT) on the Point Spread Function (PSF). The Example of Poly PSF and MTF of a Doublet has an MTF analysis done at 0.5 degrees off-axis.
  • Properties of Scattered Rays/Beams
  • RAY Command Overview
    This topic describes functions of the RAY, RAYS, or RAYSET command.
  • Ray Distributions - Apodizing
    APODIZE operates immediately on all the currently selected or considered rays and, therefore, can be used before or after a TRACE. USERAPOD applies only to rays when they are created and just for the combinations described in this topic.
  • Ray Splitting Overview
    The ASAP ray trace algorithm can split a ray into reflected, transmitted, diffracted, and scattered rays at an object interface. Not only will ASAP split the ray at an interface, it will properly adjust the flux of that ray at the interface according to the interface optical properties.
  • Ray Tracing Overview
    Ray tracing is accomplished by the RAY and TRACE commands. The fundamental difference between these commands is the permanence of the rays.
  • Referencing Entities
    Databases of surfaces, edges, objects, media (including coatings), rays, and sources may be referenced absolutely or relatively. Objects, media, and coatings also may be referenced literally.
  • Refraction/Reflection Controls
