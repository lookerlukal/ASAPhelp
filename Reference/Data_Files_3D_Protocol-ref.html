3D Data Files (Protocol)

The protocol for the three-dimensional (3D) data files is a simple one devised by BRO. Each line in the file corresponds to a coordinate in 3D space.

CODE,X,Y,Z [,A,B,C] [ /identifier ]

X,Y, and Z are the floating point coordinates in system units. A,B, and C specify an optional associated vector (usually a surface normal). They will have to be converted to pixels, inches, etc. for your particular device. The integer part CODE parameter is defined as follows:


Integer Part of CODEAction
0Move to X,Y,Z no drawing
> 0 Draw a dot at X,Y,Z
< 0Draw a line to X,Y,Z

The actual CODE value assigned to each point is application dependent but is usually assumed in most BRO software to be an integer color number with a fractional object number.
