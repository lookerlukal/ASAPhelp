CODE V to ASAP Translator Restrictions

While this translator supports most of the optical system modeling capabilities of CODE V, there are inevitable differences in the database structure and intended applications, so that a precise translation is not always possible. In most cases, the equivalent functionality can be accomplished in ASAP with user intervention. Specific features and limitations of this translator are described in this topic.

Afocal systems:

AFC is supported but accommodation ACM is not.

AFO is not supported. The perfect lens is modeled as a dummy plane.

AFI is supported. The perfect lens is modeled as an IDEAL ASAP lens.

Alternate surface intersection: Alternate surface intersection flag IC is not supported.

Apertures: Only simple circular, elliptical, and rectangular apertures are supported. Aperture decentrations are supported but aperture rotations (ARO) are not. If there are multiple apertures on a surface, only the first aperture is used. Labeled apertures are supported.

Array elements: Array elements are not supported.

Coatings: Coatings are not supported.

Depth of focus: Only the nominal focal plane location is translated.

Fictitious glasses: CODE V uses a proprietary dispersion model based upon six “real” glasses to compute the refractive indices of fictitious glasses at the specified wavelengths. A similar model was developed for this translator; however, there are small refractive index differences. BRO recommends that you replace fictitious glasses with “real” glasses prior to attempting a translation.

GDP: This partial dispersion option is not supported.

Gradient index (GRIN) media: Only the reference wavelength gradient function is in the ASAP model. The following gradients are not supported:

Gradient Lens Corporation

Luneburg and Maxwell

Selfoc

User-defined gradients

Holographic surfaces:

Volume HOEs are not supported; they are translated as surface HOEs.

Phase terms involving absolute values are not supported. These include HCT AXY, HCT XAY, HCT XYA and HCT AR.

User-defined phase functions are not supported.

CODE V can model a two-point HOE with aspheric phase terms; however ASAP has no equivalent diffractive entity. The translator breaks the single CODE V HOE surface into two (or three) ASAP objects a small distance apart. The first ASAP object is the two- point HOE; the second is the aspheric phase HOE. (If the CODE V surface is reflective, a dummy reflecting object is also created to restore the propagation direction.)

You must set the SEARCH command so that the rays follow the desired trajectory.

Interferometric Deformations and Intensity Apodizations are not supported.

Lens modules: Only the focal length and principal reduction ratio are supported.

Non-sequential surfaces: While ASAP raytrace is fundamentally non-sequential, the differences between CODE V and ASAP make an automated translation unfeasible. Consequently, predefined non-sequential modules such as corner cubes (CCR) and roof prisms (ROO) are also not supported.

Non-spherical surfaces:

Thermal gradient surfaces and user-defined surfaces are not supported.

Pickups: Pickups are not supported. This is not an issue with CODE V-generated SEQ files, since CODE V writes both the pickup instructions and the resulting parameter values into the SEQ file. However, user-generated SEQ files must contain the resulting parameter values for the translation to be successful.

Polarization: Polarized sources, linear polarizing elements and retarders are not supported.

Private catalog:

Dispersion formulae are not supported.

Melt fit option MLT is not supported.

Transmission data TRN , specific gravity data SPG, and price data PRC are not supported.

Pupil Apodization: Pupil apodization is not supported.

Refract/reflect mode: RMD is not supported.

Solves: Solves are not supported. This is not an issue with CODE V-generated SEQ files, since CODE V writes both the solve instructions and the resulting parameter values into the SEQ file. However user-generated SEQ files must contain the resulting parameter values for the translation to be successful.

Special surfaces:

Bounding box dimensions are not used by the translator.

CN1 surfaces (Fresnel surface on a right circular cone with parallel back cuts) are not supported.

CN2 surfaces (Fresnel surface on a right circular cone with non-parallel back cuts) are not supported.

FRS surfaces (Fresnel surface on planar substrate) are not supported.

SEG surfaces (segmented conic surface) are not supported.

XYP polynomial surfaces are not supported.

ZRN surfaces (standard Zernike surface) are not supported.

Specification data: Image space specifications (for example, FNO, XIM) are not supported. Operating modes such as WID, CRA, TEL, XZF do not affect the system geometry and so they are not translated. You must construct the desired raytrace characteristics.

Surface grouping: Surface grouping is not supported.

Thermal expansion, atmospheric and pressure data: Environmental conditions are not supported.

Tolerance data: Tolerance data is ignored by the translator.

Wavelengths and refractive indices: The translator writes out all the wavelength and refractive index data that is specified in the CODE V SEQ file. Make sure the number of wavelengths/refractive indices does not exceed the ASAP capacity.
