Reference (D-F)

Reference topics in this section begin with the letters "D-E-F".

  • 2D Data Files (Central Plot Routine Protocol)
    The two-dimensional (2D) plot data follows very closely the CALCOMP protocol. Each line in the file corresponds to a call to the PLOT routine and contains essentially the calling arguments.
  • 3D Data Files (Protocol)
    The protocol for the three-dimensional (3D) data files is a simple one devised by BRO. Each line in the file corresponds to a coordinate in 3D space.
  • Data Storage for Entities
    SURFACE/FUNCTION , EDGE/CURVE and LENS data currently share the same array for internal storage. In fact, they have equivalent storage locations.
  • DECOMPOSE Command Overview
    Aperture diffraction calculations made by ASAP are partly a function of the beam sampling of the aperture. The question is whether a suitable number of Gaussian beams clipped by the aperture so that the aperture apodization is represented in the subsequent diffraction pattern.
  • DECOMPOSE with a Source
    When using the DECOMPOSE command, treat it as a method for creating the spatial and angular properties of the source. You must, of course, still define the appropriate physical properties of the source.
  • Defining a BSDF Model
    This topic illustrates how to define a BSDF model in ASAP.
  • Defining Interpolation Functions
    The one- and two-dimensional interpolation functions (F1LINTERP and F2LINTERP) are found in the source code. These routines are used only locally (that is, by the user-definable routines), and you can modify them.
  • Defining Nondispersive MEDIA
    In some situations, you can safely ignore the polychromatic nature of light to perform a valid systems analysis.
  • Defining an Apodization of Emitting Volumes
    This topic lists the input arguments and return for defining an apodization of emitting volumes.
  • Defining a Positional or Directional Apodization
    This topic lists the input arguments and return for defining an positional or directional apodization.
  • Defining a Surface or Function
    This topic lists the input arguments and return for defining an surface/function.
  • Defining a Surface Intersection
    This topic lists the input arguments and return for defining an surface intersection.
  • Defining a Surface or Object Sag
    This topic lists the input arguments and return for defining an surface intersection.
  • Defining WAVELENGTHS and Associated Dispersive MEDIA
    This topic lists the input arguments and return for defining WAVELENGTH(S) and associated dispersive MEDIA.
  • DEFORM with Arrayed Sampled Surface
    The example, Deforming to an Arrayed Sampled Surface, demonstrates not only the DEFORM command, but also several other commands and useful techniques for modeling complicated objects.
  • Diffraction Calculations
    The technique ASAP uses for diffraction analysis is named beam superposition. While superposition theory is not an entirely new concept, the combination and implementation of it with real ray tracing is new. The ASAP technique is substantially enhanced to include: effects of higher-order Gaussian beam modes, polarization, and general electromagnetic wavefront propagation, including mixed coherent/incoherent spread functions.
  • DISPLAY - Inputting an Arbitrary Data Distribution
    Creating a user-defined distribution of data for plotting.
  • Displaying Graphics in 2D or 3D
    ASAP has complete 2D and 3D graphics capabilities that include overlays, oblique views, annotation, static and dynamic rendering, zoom in/out, interactive point-and-shoot ray tracing, and many other features.
  • Distribution Data Files Overview
    ASAP has several commands for calculating the irradiance, intensity, radiance etc., for a set of files, some of which produce immediate results of these calculations in the form of plots directly on the screen.
  • Distribution File Structure
    Distribution files are unformatted (binary) direct access files with a five-record header followed by the data array.
  • Distribution Data Display
    Many analysis commands (SPOTS, OPDMAP and others) create distribution data files. These files contain matrices of various forms of data (flux data, optical path difference data and others) and are stored on disk in binary format for efficiency as they can become quite large.
  • Distribution Data Files - Data Manipulation
    ASAP supports a rich set of data processing commands that act on the distribution data files.
  • DOE Lens
    The surfaces created by the MULTIPLE command are the key to modeling diffractive optical elements (DOE). You must create a multiple surface whose intersection with the base surface specifies the grating lines.
  • Dynamic Rendering
    The ASAP 3D Viewer is linked to the OpenGL libraries for real-time, dynamic rendering of systems and ray traces.
  • Edges and Curves Overview
    Edges and curves are made up of a series of connected points in space that form a parametric representation of an entity.
  • EMITTING Source Overview
    The following commands randomly create rays within a specified volume. The flux of each ray can be modified via an "apodization" function programmed by the user USERAPOD ANGLES/BOTH commands, EMITTING CONE/PYRAMID and EMITTING BOX/SPHEROID.
  • Examples of Command Scripts
    You can open any one of the hundreds of example scripts in the form of ASAP INR files. These scripts are accessible in three ways: from the command help topics, from your install directory, or directly from the Quick Start toolbar. Select the Example Scripts tab on the toolbar, and then select the command name or example file name you are interested in viewing. More than one example may be listed for a command or category of commands. The script will open in the Editor window.
  • Extended Sources - Setup and Analysis
    This topic describes how to set up sources for analysis.
  • Files Produced by ASAP
    Lists the file types that ASAP produces and uses.
  • File Structure of ASAP
    ASAP is a sophisticated set of 3D system modeling and optical ray/beam propagation algorithms. It performs the entire ray tracing and derivative calculations.
  • Finite-difference Wave-field Propagation
    ASAP OPTICAL+ uses a unique finite-difference beam-propagation method (FD-BPM) to carry fields into the realm of single-mode fibers, splitters, couplers, and other guided-wave problems.
