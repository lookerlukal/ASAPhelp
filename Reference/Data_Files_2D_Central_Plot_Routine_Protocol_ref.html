2D Data Files (Central Plot Routine Protocol)

The two-dimensional (2D) plot data follows very closely the CALCOMP protocol. Each line in the file corresponds to a call to the PLOT routine and contains essentially the calling arguments.

CALL PLOT(X,Y,IPEN) -99.999 < X < 99.999
-9.999 < Y < 99.999 (maximum)
0. < Y < 11. (typical)

Except for an IPEN value of 0, X and Y, the floating point coordinates are in inches in the long and short directions of the device, respectively. If the origin is not changed, X can take on values from -99.999 to 99.999, and Y typically from 0.000 to 11.000 inches. IPEN is the integer "pen" control parameter and can take the following values:


IPENACTION
0Set character height to X inches and rotation to Y degrees
1Move to X,Y with pen up, put pen down (simple point plot).
2Move to X,Y with pen down (draw line).
3Move to X,Y with pen up (no drawing, simple move).
4Fill a rectangle from last point to diagonal corner at X,Y.
5-31Change pen attribute* to IPEN-4, and then move to X,Y with pen up.
32-127Place corresponding ASCII character at X,Y (lower left corner).
128-998Reserved for other attributes and/or character fonts.
< 0Perform above functions, and then place new origin at X,Y.
999End of plot and dget ready for new plot indicator (exit graphics mode; set flag to enter graphics mode, and clear device on next write to plot X and Y not used.

*The pen attribute is assumed to be a color number from 1 to 27 with 1 being completely opposite of the devices background color (the default) and 27 being the same as the background. For example, a useful default definition for the colors in terms of relative primary content would be:
