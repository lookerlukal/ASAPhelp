Optimization Exit Criteria Tab

The Exit Criteria tab on the Optimization Setup Summary window lets you select the exit criteria that determines completion of optimization.

You can switch to another tab on the Optimization Setup Summary window without performing final validation. Some validation may occur when you select on the Optimization Method tab.
Property Type Field Property Description
Limit on Number of Trial Solutions Required integer Maximum number of Trial Solutions that will be evaluated in the search for an optimal design. The number must be greater than zero (>0). Default is 100.
Use Acceptance Limit Optional real value The number must be a positive real value. If this field is clear, the default is none. If this field is selected, the default is 0.0010.
Use Relative Divergence of Trial Solutions Optional Provides a way to terminate optimizations that are diverging in excess of a user-specified, acceptable rate. If this check box is selected, the following two fields are required: Relative Divergence Limit Ratio and Number of Consecutively Preceding Trial Solutions.
Relative Divergence Limit Ratio Required The value for the ratio r entered represents the acceptable ratio of the FOM over any of the n consecutively preceding trial solutions as a condition for considering additional trial solutions. If the check box, Use Relative Divergence of Trial Solutions is selected, the default value of 0.5 is displayed. The default can be changed.
Number of Consecutively Preceding Trial Solutions Required The value for n represents the number of consecutively preceding trial solutions used to evaluate the rate of divergence. If the check box, Use Relative Divergence of Trial Solutions is selected, the default value of 10 is displayed. The default can be changed.
Use Acceptable Noise Optional Provides a way to terminate optimization after an acceptable "signal-to-noise ratio" is achieved. If the check box is selected, the following two fields are required: Acceptable Noise Ratio and Number of Consecutively Preceding Trial Solutions.
Acceptable Noise Ratio Required The value for the ratio r represents the maximum variability over a number n of consecutively preceding trial solutions as a condition of considering additional trial solutions. The acceptable noise ratio is the "inverse signal-to-noise ratio". If the check box, Use Acceptable Noise is selected, a value, the default value of 0.01 is displayed. The default can be changed.
Number of Consecutively Preceding Trial Solutions Required The number of consecutively preceding trial solutions, n is used to evaluate the "signal-to-noise ratio". If the check box, Use Acceptable Noise is selected, the default value of 10 is displayed. The default can be changed.
Close Button Closes the Optimization Setup Summary window, and ends the Optimization session.
Help Button Opens Help topic that describes the dialog box.
