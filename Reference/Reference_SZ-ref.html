Reference (S-Z)

Reference topics in this section begin with the letters "S" to "Z".

  • Scattering Overview
    Scattering is generally defined as the removal of energy from an incident wave and the subsequent re-emission of some portion of that energy. The macroscopic laws of refraction and reflection may even be considered special cases of the statistical averages of scattering phenomena. This topic discusses scatter from nonsmooth surfaces; that is, surface scattering, since ASAP is primarily designed to model surface scattering phenomena. Volume scattering can be modeled by using many transparent scattering surfaces to partition the volume.
  • Simulating Diffractive and Holographic Optical Elements
    Diffractive Optical Elements (DOEs) are a general class of optical elements that include diffraction gratings, holographic optical elements (HOEs) and binary optics. These elements diffract light by producing periodic changes in the phase or amplitude of an incident wave. This topic addresses DOEs that change the phase of incident optical fields.
  • SPREAD and FIELD Command Differences
    The two field/energy distribution commands, SPREAD and FIELD, appear to perform similar calculations (and in fact can perform many of the same calculations), but there are differences described in this topic.
  • SOURCE Command Overview
    The SOURCE command establishes the ray propagation direction and its initial optical path length. Use it in conjunction with GRID commands, which establish the initial ray coordinates. You can also use this command to create grids of sources.
  • Spectral Weighting
    Most polychromatic sources (and detectors, depending on the view of the problem) do not radiate equally across their spectral bandwidth. Since polychromatic sources in ASAP are made up of individual monochromatic sources, the FLUX command may be used to spectrally weight the individual monochromatic sources.
  • Statistics for SUMMARY
    The SUMMARY command, a subcommand of HISTORY, is described in this topic.
  • Stray Light Analysis - Identifying Critical Objects
    Identifying critical objects is perhaps the most important part of a stray light analysis. The critical objects are those objects seen by the detector, either directly or directly imaged onto the detector.

  • Starting Sources in Arbitrary Media
    ASAP initializes MEDIA 0 to represent vacuum/air, so you do not need to define a medium.
  • USERAPOD Commands Overview
    The USERAPOD command imposes either a spatial or directional apodization (both options can be used together) on a distribution of rays.
  • User-Definable Routines
    With ASAP, you can create user-definable routines for surface intersection, surface functions, BSDFs, and other features. ASAP is dynamically linked with a library (userprog) of user-definable routines. The library contains routines that ASAP uses for surface intersections, surface functions, BSDFs, and other functions.
  • Volumetric Energy Tracking
    The space containing all or part of an optical system can be divided into volume elements or “voxels” to analyze and visualize volume absorption, fluence, and irradiance within this space.
