CODE V to ASAP Translation, Step 1

The first step in the translation process involves selecting an *.SEQ file written by CODE V. You can either type the name of the file in the box below Select CODEV Lens (sequence) File, or click BROWSE. Either of these actions will display a FILE OPEN dialog, from which you can select an SEQ file.

Note: The translator supports only SEQ files for translation. CODE V also writes *.LEN files, which are binary files using ORA’s proprietary format. These files cannot be translated. User-generated SEQ files may be translated provided the data is arranged in a similar fashion to a CODE V-generated SEQ file. )

The translator confirms the file’s existence by attempting to open it. If it is successful, the name of the selected file appears in the edit box, and the translator creates a default name for the ASAP file. (In this example, the default name for the ASAP file is wideangle.inr.) If the translator cannot open the SEQ file, it displays a message box. The translator cannot move to the next dialog without a valid filename.

After you select the CODE V SEQ file, the translator reads the file for compatibility with ASAP and displays a summary in the scrolling list box. The list box has four columns of adjustable width so that you can read any text that might be truncated. The columns are:


Surface The surface number for which the translator is commenting.
Command The exact text of the CODE V command that the translator read.
Message A brief message summarizing the translator’s response to the CODE V command.
Explanation If the command is an unsupported CODE V command, this column describes the command.
Note: Messages do not necessarily indicate that a problem exists. These messages are intended to alert you to features and capabilities present in one program that may be imprecisely translated into the other.

After creating the list, the translator writes out a summary recommendation or warning. The five possibilities are:

Unknown material encountered; correct SEQ file or manually add to ASAP INR file

The translator searches through the ASAP glass catalogs to locate materials found in the SEQ file. However, since the CODE V materials database contains catalogs and materials that are different from those in ASAP, the translator may not find a given material. CODE V also supports an alternate glass name syntax (for example, 517642_SCHOTT is equivalent to BK7_SCHOTT) that the translator does not recognize.

CAUTION: The above message is serious and indicates that the translated file will most likely not run successfully in ASAP.

Special surface types and/or materials were found; check ASAP model

The translator noted these surface types while reading the SEQ file. Carefully review the notes in the list box and the ASAP model for potential problems.

Missing apertures on some surfaces; correct SEQ file or set default in next dialog

Since precise aperture information is essential to ASAP, you must supply this information to perform the translation. While it is probably best if you return to CODE V and properly set the aperture(s), you can also assign default apertures or ask to be prompted for apertures in the second translation dialog.

Unsupported commands were found and could affect translation accuracy

The translator noted these commands while reading the SEQ file. Carefully review the notes in the list box and the ASAP model for potential problems.

CODE V lens is ready for translation

No warnings of any type were issued.

At the completion of this dialog, click NEXT to continue to the next dialog, or CANCEL to exit the translator.
