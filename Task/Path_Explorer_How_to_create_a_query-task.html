Path Explorer - Creating a Query

This topic describes how to create a query in the Path Explorer.

Pre-requisites:

You must issue a SAVE command before the TRACE command in the script of your INR file or Command Input window to create a HISTORY file. The Path Explorer uses this file.

To create a query from start to finish using the Path Explorer Query Builder dialog:

  1. On the main ASAP toolbar, click the Query Builder button. If it is not on the toolbar, right-click the toolbar, select Customize, and add the button. Alternatively, you can select Query Builder from the Analysis menu. The Path Explorer Query Builder dialog is displayed for the current ASAP Path Explorer kernel session.
  2. Click the Query tab on the Path Explorer Query Builder dialog, select the query type, and complete all available fields that are dependent upon the selected comparison type.
  3. Click the Run Query button. The results of the path query are saved to an active path-set.
  4. Repeat steps 1 to 3 to create additional queries.
    Note: Each query acts by default on the active path-set, unless the GLOBAL modifier is used, and updates the active path-set with its results, unless the optional setname for the results is provided. If the GLOBAL modifier is used, the query acts on all paths, instead of the active path-set. If the optional setname is provided, the results of the query are written to that setname and the active path-set is not updated.
  • Path Explorer Query Builder/ Query Tab
    Use the Query tab dialog box to create queries of path-sets.
  • Path Explorer INTERSECTS-volume Entry Dialog
    The INTERSECTS-volume Entry dialog is displayed when you select the INTERSECTS-volume query type from the Query tab on the Path Explorer Query Builder dialog.
