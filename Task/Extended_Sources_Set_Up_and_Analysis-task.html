Extended Sources - Setup and Analysis

This topic describes how to set up sources for analysis.

Pre-requisites:

Prior to defining the rays, specify the following input in this order:


WAVELENGTH(not required)
CLIP(not required)
USERAPOD ANGLES(user-defined apodization)
POLARIZATION(emitting raysets are randomly linearly polarized (default))

The following input specifies the rayset:

EMITTING. . .

The EMITTING family of commands defines rays that are randomly distributed in position and direction over a surface or volume.

To modify the characteristics of the rayset after specifying the EMITTING commands, specify the following input:

FLUX (used to modify fluxes of rays)

To specify where and at what resolution to do the calculation, enter the following input:


CONSIDER/SELECT(not required unless you want to restrict the ray data)
WINDOW(required so that ASAP knows where to do the calculation)
PIXELS(not required unless you want higher resolution)

After following the order of the previous commands, run SPOTS POSITION, SPOTS DIRECTION, or RADIANT.

After the calculation, use DISPLAY to examine the distribution data.

Note:
  • Incoherent geometric extended sources have similar properties to point sources. However, the spatial and angular properties of the source are defined with a single EMITTING command.

  • Coherent/Incoherent geometric point sources typically are created on 2D planar grids (with a GRID command), which define the spatial extent of the source (spatial property). The grids may be rotated, shifted or otherwise transformed with the linear transformation commands. A grid of rays that has direction cosine vectors assigned to it with a SOURCE command is the actual source.

    In fact, a grid of rays cannot be traced until the direction cosine vectors assign the propagation direction to the spatial arrangement of rays (angular property). Direction cosines are geometric cosines (the sum of the squares of the direction cosines is equal to one). They are not optical cosines used by other lens design codes.

    The main check ASAP uses to determine if a set of beams comprising a monochromatic point source is coherent or not is a modification of the BEAMS command. If the BEAMS COHERENT DIFFRACT is specified, then, in general, the beams of a monochromatic point source are assumed to be coherent with respect to each other.


    • If the individual beams are coherent with respect to each other, ASAP will linearly superpose the electric field amplitudes of the individual beams to calculate the energy distributions.
    • If the individual beams are incoherent with respect to each other, ASAP will linearly superpose the modulus squared of the electric field amplitudes (energy density) of the individual beams to calculate the energy distributions.
    • Beams comprising a monochromatic point source are coherent with respect to each other. Beams comprising a polychromatic point source are incoherent between wavelengths, but coherent with respect to each other at a single wavelength.
