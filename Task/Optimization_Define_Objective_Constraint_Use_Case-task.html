Optimization - Defining Objective Constraints

This topic describes how to define and enable objective constraints for optimization. It also describes how to define and assign penalty functions. Objective constraints and penalty functions are optional.

Pre-requisites:

An open INR file that contains optimization elements, such as a scalar real variable, must be open in the ASAP Editor window.

  1. Select text from an open INR file that contain a scalar real variable and right-click the text. The selected text is a valid expression (in the form name=value, where value is an expression) that can be evaluated to create an objective constraint.
  2. Select Define Objective Constraint from the context menu. Alternatively, from the Optimize menu on the ASAP toolbar, select Define Objective Constraint.

    A new objective constraint displays on the Objective Constraints tab of the Optimization Setup Summary window. The name is automatically inserted, based on user selection in the script.

  3. If desired, define a penalty function that you can later assign to an objective constraint.
    1. Select text from the script in the Editor containing a scalar variable that can be evaluated as a penalty function.
    2. Select Define Penalty Function from the context menu. Alternatively, from the Optimize menu on the ASAP toolbar, select Define Penalty Function.

    Selected text should contain the name of the penalty function and its value; for example, P1=2, P2=1/(C+.001), and P3=5.
  4. To assign a defined penalty function to a defined objective constraint, select the name of the constraint on the Objective Constraints tab, and select a pre-defined penalty function name from the drop-down menu in the Penalty Function column on the same row.

     For example, select the objective constraint, "C", from the Name column on the Objective Constraints tab. Open the Penalty Function drop-down menu on the same row, and select the name, "P1".
    Note: You can define penalty functions before or after defining objective constraints, but a penalty function can be assigned only after an objective constraint is defined.
  5. To unassign (remove) a penalty function from a constraint, select No Penalty Function Assigned from the drop-down menu in the Penalty Function column on the same row as the constraint.
  6. Optionally, assign or change the minimum and maximum values, and enabled state for each constraint.

    Note: A constraint is applied only when it is enabled. Disabled constraints are ignored.
Result:

The objective constraint is validated and the objective constraint count is incremented.

Post-requisites:

Define valid design variables, design objectives, objective constraints, exit criteria, and optimization methods, and select on the Optimization Method tab.
