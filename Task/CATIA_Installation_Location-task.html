CATIA Module - Installation Location

Installation of he CATIA Module for ASAP requires you to specify the valid location of the Dassault Systemes bin folder and the release version on your machine.

Depending on the installation, this location is typically in C:\Program Files\DASSAULT SYSTEMES\BXX\intel_a\code\bin, where XX is a two digit number associated with the specific CATIA release. The CATIA configuration in CAD+ currently supports three CATIA release versions: R13, R14, R15. No other versions are supported by the CATIA configuration.

Three cases exist for specifying the path for the DASSAULT SYSTEMES bin folder:


  1. No location specified for importing CATIA files into ASAP.

    You are prompted via the CATIA Versioning dialog box to locate the DASSAULT SYSTEMES bin folder associated with the version you want to specify.


    CATIA Version The drop-down box contains a list of CATIA release versions that ASAP supports. The list contains strings in the form of RXX.XX = 13,14,15.
    CATIA Directory LocationEnter the CATIA bin folder location in this box, or click the ... button to open the Browse For Folder dialog. Look for the DASSAULT SYSTEMES folder and select the bin folder associated with the version you want to specify. For example, if you want version B13, go to DASSAULT SYSTEMS\B13\intel\code\bin.
  2. A previously specified location exists.

    If you previously specified a location for the bin folder within ASAP, no action is required.

  3. The location of the CATIA bin folder has changed.

    If you have upgraded to a more recent CATIA release (for example, from R14 to R15), you need to update the valid path to the new bin folder under R15 before importing CATIA files into ASAP.

    To reset the path, click the Reset CATIA Version button on the CATIA Setup Parameters dialog box in ASAP, as shown below.

Tip

BRO recommends that you do not select earlier releases of CATIA after you have upgraded to a later release.

Note:

You may be prompted to restart ASAP for the changes to take effect. Typically, a restart is required if a CATIA part was imported in the current ASAP session.

The paths and release version you select are validated during the import to assure the CATIA version and the release version are consistent. For example,

C:\Program Files\DASSAULT SYSTEMES\B15\intel_a\code\bin and release version R15 are consistent.

C:\Program Files\DASSAULT SYSTEMES\B13\intel_a\code\bin and release version R15 are not consistent.
