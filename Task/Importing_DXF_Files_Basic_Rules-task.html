Importing DXF Files - Basic Rules

For importing, use the Microsoft Command Window utility, Dxf2asap.exe (located in the asap\bin folder).

When importing DXF files into ASAP, apply these basic rules:

Transfer only objects made of edges.

Do not import SINGLE edge objects. You need at least two edges connected together to form each object.

Do not put any breaks in the edges. In other words, do not end one edge and then restart it again with a gap in between (in any one object). The points must all join together.

Assign the same number of points to all edges in an object. For example, if you have two edges making up an object, with one edge made up with six points, the other edge must also have 6 points.

Note: ASAP Lite
  • Supports the DXF translator up to release 12 of AutoCAD, but not beyond this release.
  • Supports only 3D faces and polyline meshes, which are not true representations of surfaces and yield less-accurate results over true nurb-based surfaces, such as those translated via IGES/ASAP.
  • Does not support 3D solids or B-splines.
If most of your geometry is being done in CAD, BRO recommends you use the IGES/ASAP translator.
