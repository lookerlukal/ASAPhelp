Ray Tracing - Improving Speed

The ray trace algorithm used in ASAP is optimized for speed and accuracy. This topic describes how you can speed up the process.


  • Trace only as many rays as are needed. There are many instances when a single ray may provide sufficient information about the properties of a system.
  • Use LENS/OBJECTS where appropriate. Since rays must propagate through lens conicoids sequentially, ASAP is not slowed down by searching.
  • Use SEARCH to optimize the search strategy. This method may require modifying the input so that the objects are numbered in a particular manner.
  • Do not use PARABASAL rays unless you need them; they are traced as rigorously as base rays.
  • Turn ARROWS OFF to keep the plot files smaller (ray plot data is only one/fifth as large with ARROWS OFF ).
  • Do not do a TRACE PLOT LIST unless you need to because it causes excessive disk I/O.
  • If you are solely interested in screen graphics and do not want to save the graphics files, then turn off the 2D graphics file with $IO PLOT CANCEL , and turn off the 3D vector file with $IO VECTOR CANCEL .
