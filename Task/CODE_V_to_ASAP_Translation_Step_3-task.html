CODE V to ASAP Translation, Step 3

The final step in the translation process is to choose the zoom position(s) to translate and, optionally, the surface for the origin of the global coordinate system.

The message under Refractive media reminds you to verify the MEDIA in the ASAP file, particularly if an unknown material was encountered.

The translator offers four options for writing the ASAP INR file:


  • Translate only first position Translates only the first zoom position. If the system is not zoomed, the first position is the only position. The ASAP filename is the name selected in the Step 2 translation dialog. (Default: selected)
  • Translate only zoom position XXX Translates only the specified zoom position. Appends _ZOOMXXX to the ASAP filename that was selected in the Step 3 translation dialog, where XXX is the number of the zoom position. If you enter an invalid zoom position, the translator issues a warning message. (If the system is not zoomed, any entry other than 1 produces a warning message.)
  • Translate all zoom positions Writes out a separate ASAP file for each zoom position found in the SEQ file. Appends _ZOOMXXX to the ASAP filename selected in the Step 2 translation dialog, where XXX is the number of the zoom position.
  • Translate all zoom positions such that the origin is at surface number YYY Writes out a separate ASAP file for each zoom position found in the SEQ file. Appends_ZOOMXXX to the ASAP filename selected in the Step 2 translation dialog, where XXX is the number of the zoom position. See Note below.
Note:

The significant functionality of the last option is the ability to specify surface YYY as the origin of the global coordinate system. By default, the translator makes the object surface the origin of the global coordinate system in a finite conjugate system and the first non-dummy surface the origin in an infinite conjugate system. However, if you are combining several translated files into a single ASAP INR file, it may be expedient to specify some other surface as the global origin so that you can “cut and paste” the composite model together.

Furthermore, this option is also available to non-zoomed systems for those instances where the default global origin is not convenient.

At the conclusion of the translation, the translator displays a message box showing the location of the original CODE V file and the translated ASAP file.



If the translator created more than one ASAP file, the message box notes this and displays the range of names.



With the exception of the SPS CPC surface, all the ASAP surfaces are functional duplicates of the CODE V surfaces. However, in the case of the SPS CPC surface, where there is no equivalent ASAP surface, the translator prompts you for a number of sample points. The translator generates your specified number of linearly-connected points on the curve, extrudes or sweeps the curve (depending upon the type of CPC) to create a close approximation to the CODE V surface, and then writes the points out as an EDGE-based object.



You can edit the translated file and add the SMOOTH command, if desired, to smooth out the linearly-connected curve. However experiments conducted during the development of the translator indicate that agreement with CODE V is best obtained with 300 to 500 linearly connected points.

At the completion of this Step 3 dialog, you can click BACK to return to the previous dialog, FINISH, or CANCEL to exit the translator.
