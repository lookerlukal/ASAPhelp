Use Model Accuracy(not available for CATIA)

Select for ASAP to import the geometry using the degree and product specified for the geometry in the file. No approximation of the underlying geometry is performed where possible to improve import accuracy and shape identity of the geometry; however, ray tracing may slow due to more complex geometry.

Deselect to activate the maximum degree product and Bezier order spinner arrows, so you can choose the values. Note: deselecting this option may speed up ray trace times, but it may also adversely affect the quality of the geometry imported into ASAP.

Maximum degree-product of PATCHES

Increases maximum number of PATCHES to define the surface. See Notes below.
Locate Duplicate GeometryReports detection of duplicate surface geometry during import.
Bezier OrderMaximum order of the explicit polynomial that defines the Bezier segment (not more than 20). See Notes below.
Display Setup Dialog If selected, displays this dialog box tab upon each import.
Material Database/Use Current DirectorySelect current directory or browse to another directory to store *.dat files for the material database.
Note:
  1. The correct settings for the Maximum degree-product of PATCHES and the Bezier Order are important for two reasons: speed and accuracy. The "degree-product" is the product of the orders of the Bezier equations, defined in orthogonal directions, comprising the surface. The Bezier order is the order of the explicit polynomial. For example, a surface composed of a second order parametric equation swept along another second order equation has a degree-product of 1. Each polynomial has a Bezier order of 2.
  2. The ray trace speed is also directly related to the degree-product and Bezier order. The more patches and the higher the Bezier order, the longer the ray trace times. In fact, when the patch degree exceeds second order, ASAP issues a warning informing you to expect "Slower ray tracing ahead." The maximum degree-product is currently defaulted to 5 and the Bezier order is defaulted to third order. These settings are suitable for many illumination applications and some imaging applications. However, you should perform the analysis at different settings to access the accuracy for your particular application, especially if it is an imaging application. Translations involving imaging applications should be done at their exact orders.
  3. The translator by default builds a lofted bi-quadratic rational Bezier surface approximation to the original NURBS surface. It chooses ahead of time whether to loft in the u or v parametric direction, based on minimizing the number of ASAP CURVE entities. Sometimes the other approximation (even though it uses more CURVES) is closer to the original NURBS.
  4. If the NURBS surface in the IGES file is already in Bezier format (an export option in some CAD systems) AND its degree product does not exceed the specified value AND the maximum order (interchangeable with "degree") in either parametric direction does not exceed its specified value, then the surface is transferred directly and exactly to ASAP. However, raytrace times may increase dramatically relative to the default spline approximation discussed in 3 above).
