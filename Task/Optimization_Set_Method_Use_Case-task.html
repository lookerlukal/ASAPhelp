Optimization - Defining a Method

This topic describes how to set a predefined optimization method.

Pre-requisites:

Any suitable ASAP INR script file open in the ASAP Editor window, and you have defined at least one design variable and design objective.

  1. Select Define Optimization Method on the context menu. Alternatively, from the Optimize menu on the ASAP toolbar select Define Optimization Method. The Optimization Method tab is displayed. The tab changes depending on the method you selected.

  2. If you are using more than one variable, confirm that the selected method (Downhill Simplex or Simulated Annealing) is the one you want. If Downhill Simplex is the selected method, an example of the Optimization Method tab is

    If Simulated Annealing is the selected method, an example of the Optimization Method tab is


  3. Verify that the entered Optimization Session Title is acceptable; change it if it is not.
  4. Complete the enabled fields for the method by either accepting the defaults or changing the settings.
    Note: For example, if you select only one design variable, or have deleted defined variables so that only one remains, only Brent 's method is available as the optimization method. If you define a second design variable, ASAP automatically changes the optimization method to Downhill Simplex, which is the default optimization method. You can change the optimization method to Simulated Annealing (the only other valid choice for two or more design variables) if this method is your preference.
  5. Select . To stop optimization, select from the ASAP toolbar. If ASAP validates the data, it performs the trial solution.
    Note: If the required data is not validated, optimization is halted and an information message is displayed.
Post-requisites:

A tabular report and a Cartesian plot are generated for trial solutions.
