OBJECT Command Overview

In ASAP, an object is a bounded physical entity with optical properties. The OBJECT command associates information from the optical property databases with the system geometry databases (MEDIA, COATINGS, MODELS, SURFACES, EDGES, LENSES, and others). Objects are subsequently constructed from these databases.

Objects are defined in a single global coordinate system with each object existing as an independent entity that may be transformed (rotated, shifted, and so on) without altering the locations of other objects.

Rays/beams intersect objects and are absorbed or redirected by them (that is, the object is what is ray traced in ASAP, not surfaces, edges, or lenses).

Construction of Objects from Data Bases

The physical shape of the object is determined from the surface, edge, or lens databases (or some combination).

The physical bounds of the object are defined by:


  • Default, if the object is constructed from edge data (edges are self-limiting)
  • Default, if the object is constructed from lens data (lenses are self-limiting)
  • Default, if the object is constructed from certain surface types that are self-limiting
  • Specifying the physical dimensions of the object in either local or global coordinates
  • Specifying bounding surfaces
  • Specifying bounding surfaces to form a solid
  • Specifying edges to form a mesh
Tips

When constructing an object:


  • Issue a SURFACE, EDGE, or LENS command to tell ASAP the type of entity you are defining and to bound the entity.
  • If applicable, use a LOCAL command to bound the object.
  • Use the appropriate command for the desired entity.
  • Use an OBJECT command to make an object from the entity.
  • Use SHIFT and ROTATE to position and orient the object.
  • Use the INTERFACE command to assign optical properties.
