Ray Splitting Overview

The ASAP ray trace algorithm can split a ray into reflected, transmitted, diffracted, and scattered rays at an object interface. Not only will ASAP split the ray at an interface, it will properly adjust the flux of that ray at the interface according to the interface optical properties.

Ray splitting is possible under four conditions:

INTERFACE has a nonzero reflection and transmission coefficient.

INTERFACE is diffractive with multiple orders.

INTERFACE references a birefringent MEDIA.

OBJECT has scattering properties.

The first three conditions are specular properties of the object INTERFACE command. The fourth condition is a scattering property of the object SCATTER command.

The ability of an object to generate split rays is controlled by the SPLIT command and the parameters of the INTERFACE command. The SPLIT command controls the number of specularly split rays. The INTERFACE command references the MEDIA and COATING databases. It therefore controls whether reflected and transmitted rays are split and the flux apportioned to those rays. On diffractive interfaces, the INTERFACE command controls the type of diffractive order and the order efficiency.

Parent and Child Rays

At an interface with appropriate characteristics, the most energetic ray (referred to as the parent) is allowed to continue to propagate as it would in a conventional ray trace code. The position, direction, and optical path length, along with other ray information of the less energetic ray (referred to as the child) are written to the virtual ray storage disk file for tracing at a later time. A primary ray pointer tracks the most energetic rays so that those rays are traced first. A secondary ray pointer tracks the other split rays. When the secondary pointer catches up to the primary pointer, all original rays and split rays have been traced through the system. It is possible to continually create split rays such that virtual ray storage will fill your disk space.
