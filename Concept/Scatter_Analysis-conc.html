Scatter Analysis

Performing a scatter analysis is a complex task. In addition to basic scatter theory, there are many ASAP parameters to consider. Each analysis should be evaluated on an individual basis to determine which ASAP tools to use to simulate the problem.

A fundamental concept to consider first is the type of scatter you want to simulate:


  • In-field scatter, also called near-specular scatter, is the scattered radiation propagating very close to the specular beam. It is typically generated from the specular beam interacting with scatter surfaces such as lenses and mirrors.
  • Out-of-field scatter, conversely, is scattered radiation that does not propagate close to the specular beam. In fact, it may be incident radiation from beyond the FOV of the optical device that scatters off baffles or mounts and eventually reaches the detector or image plane.
ASAP Scatter Commands

ASAP uses different scattering commands to simulate in-field and out-of-field scatter as well as different analysis techniques to analyze the different scatter contributions.

In either in-field or out-of-field scatter, ASAP generates incoherent scattered rays at an object in a similar sense to generating specular split rays. The OBJECT command modifier, SCATTER, informs ASAP that the object is a scattering object. SCATTER is an independent modifier of the object, but it can also use information from another optical properties database, called MODELS, which contains scatter models.

The OBJECT modifier distinction is closely associated with the type of scatter simulation you are performing. For example, SCATTER BSDF or SCATTER RMS are OBJECT modifiers for in-field scatter calculations. SCATTER RANDOM or SCATTER MODEL i are OBJECT modifiers for out-of-field scatter calculations. For more information, see Scatter Analysis: Common Approaches.

Specular Split Rays

When generating specular split rays, the SPLIT command controls the number of splits. An analogous command, LEVEL, controls the number of levels of scattered rays. For in-field scatter calculations, LEVEL sets to 1. At each interface, ASAP generates a scattered ray that travels along with the specular ray. Higher levels of scatter, for in-field scatter from optically polished surfaces, are unnecessary: the power scattered by a scattered ray is significantly less than the original signal ray power.

Out-of-Field Scatter

The LEVEL command is typically set to a larger number for out-of-field scatter calculations. In these situations scattered rays may contain high enough power levels to be of significance at the next level of scatter. Also, the SCATTER RANDOM or SCATTER MODEL object modifiers allow multiple rays to be generated for a single input ray, independent of the LEVEL command setting, which is necessary to simulate radiation propagating in directions other than the specular direction, that is, out of field.

Out-of-field scatter calculations require generating many rays. The rays are generated with a Monte-Carlo technique whereby the ray direction is randomly chosen to propagate into a hemisphere, and the ray flux is adjusted according to the BSDF of the object.

Although all the scattered rays can propagate into the hemisphere, but this is an inefficient simulation technique. For example, it requires generating a large number of rays for each incident ray to place enough rays at the detector to obtain a reasonable statistical accuracy for the simulation.

Improving Scatter Calculation Efficiency and Statistical Accuracy

To improve the scatter calculation efficiency and statistical accuracy, ASAP uses a modified Monte-Carlo technique to force scattered rays to propagate toward important areas. It is the TOWARDS command - a SCATTER RANDOM and SCATTER MODEL modifier.

Analysis

The results of in-field and out-of-field ray trace calculations are analyzed with different techniques. The scattered rays generated for in-field calculations are incoherent beams with a specific set of propagation rules. SPREAD NORMAL analyzes their combined effect. Scattered rays generated for out-of-field calculations are simulated as rays and not beams. SPOTS analyzes their combined effect. SELECT and PATHS commands isolate beams of interest for your analysis. Whereas in-field calculations are almost automatic, out-of-field calculations require more user interaction. You must define the appropriate scatter models, create important areas for an efficient scatter ray trace, and then analyze the results with a SPOTS command.
