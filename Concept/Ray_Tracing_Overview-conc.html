Ray Tracing Overview

Ray tracing is accomplished by the RAY and TRACE commands. The fundamental difference between these commands is the permanence of the rays.

Note: The RAY command is commonly used for diagnostic purposes, while the TRACE command is used more generally.
  • Ray Trace Data
    The STATS and LIST commands can provide you with specific ray trace information on an OBJECT by OBJECT basis or on a ray-by-ray basis, respectively, which may help you understand the propagation of energy within the opto-mechanical system under analysis.
  • Ray Tracing Data Saved by ASAP
    For each ray traced by ASAP, types of data can be saved in ASAP (in the virtual.pgs file), depending on the XMEMORY setting and your particular edition of ASAP. This topic lists these types of data.
  • Ray Tracing - Improving Speed
    The ray trace algorithm used in ASAP is optimized for speed and accuracy. This topic describes how you can speed up the process.
  • Ray Tracing Tips
    ASAP begins tracing rays from the point at which they were defined. Inappropriate starting points can give rise to unexpected results. This topic provides tips on ray tracing.
  • Ray Tracing Utility Commands
    Commands that toggle flags for the ray trace algorithm, set options, thresholds and others, are referred to as utility commands. A list of these commands is included in this topic.
