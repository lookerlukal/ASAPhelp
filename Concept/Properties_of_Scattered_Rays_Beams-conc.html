Properties of Scattered Rays/Beams

When the LEVEL command is turned on and a ray/beam hits an object with a SCATTER RMS/BSDF interface, a near-specular, forward-scattered ray/beam may be generated (subject to flux thresholds) with the following properties:


  • The scattered ray/beam has the same number of parabasal rays as the parent ray/beam.
  • The trajectory of the scattered base ray is identical to that of the specular base ray.
  • Only one forward-scattered ray/beam is generated for each parent ray/beam incident on the surface.
  • The scattered ray/beam is always incoherent; therefore the signal ray/beam generating the scattered component should be incoherent in order to conserve energy. Generating a scattered ray/beam from a coherent source can lead to energy conservation violations.
  • The scattered ray is considered unpolarized by ASAP analysis commands.
  • The beam shape is a Lorentzian rather than Gaussian.
  • The flux contained in the incident parent ray/beam is apportioned between the specular and scattered outgoing ray/beams according to the transmission, reflectivity, and the BSDF of the interface. Recall the difference in ray flux weighting between coherent and incoherent rays. The flux apportioned to the scattered ray is based upon the incident ray flux (ASAP apportions the TIS among the scattered rays). Coherent beams will therefore apportion a different flux to the scattered rays than incoherent rays. It is recommended that you create an equivalent incoherent beam (equivalent to the coherent beam but with a RECT shape factor and WIDTH 1) and scatter rays from this source in order to maintain energy conservation.
  • The coordinates of the waist rays are unchanged, but the divergence rays are altered so that they appear to originate at the intersection coordinates of the base ray.
  • Near-specular scattered rays are not allowed to rescatter at another object with a SCATTER RMS/BSDF interface.

  • If a back-scattered ray/beam is generated, it has the same properties as the forward-scattered ray/beam with the following exceptions:

    - The trajectory of the scattered base ray is back along the incoming direction of the specular base ray.

    - Only one back-scattered ray/beam is generated for each parent ray/beam incident on the surface.

    - The beam shape is a top-hat function rather than Gaussian.

    - The FIELD command ignores scattered rays; any mixed specular/scattered ray calculations must be done with SPREAD NORMAL.
