Parabasal Ray Tips

This topic discusses parabasal rays with respect to base ray trajectory, clipping, and ray intersection.

Following the Base Ray Trajectory

Parabasal rays must follow the same trajectory through the system as the base ray about which they are defined. In other words, the rays must intersect the same objects in the same order. Once a base ray has intersected an object, ASAP tries to force the parabasal rays to intersect the same object as well.

Ideally, parabasal rays always intersect the same objects as the base ray and always do so inside the physical limits of the objects. When this sequence does not happen, ASAP only performs the LIMITS and BOUNDS checking on the base ray - the parabasal rays are NOT subject to these tests, unless the PARABASAL. . .CLIP option is used. Consequently the parabasal rays cannot intersect anywhere on the base entity used to define the object intersected by the base ray.

Clipping

The parabasal rays cannot be clipped off the base ray. If the base ray misses an object, the parabasal rays also miss the object regardless of the physical geometry of the system.

Parabasal Ray Intersections

When parabasal rays cannot intersect a given object because of the functional form of the base entity, ASAP propagates the parabasal rays to the point of closest approach to the object and use that point to determine the local entity normal. There is no guarantee that the parabasal rays are heading in a useful direction after this happens.
