Optimization - Overview

The Optimization feature in ASAP addresses your need to optimize optical systems under evaluation that were designed or analyzed with ASAP. This topic gives a frame of reference, focusing on criteria you need to satisfy, and the process of designating an optimization method before you can initiate an optimization run in ASAP via the user interface. Related topics are listed at the end of this and other topics for this feature.

Purpose

Optimization is the automated adjustment of user-specified optical system parameters to improve your specified figure of merit, based on one or several objective functions. You can apply one or more prescribed strategies under your specified constraints.

In ASAP, optimization is a continuous, nonlinear, non-convex optimization on simply connected domains. You can use it as a tool to expedite product development or to explore design space.

Using the Optimization Interface

The optimization user interface in ASAP, named the Optimization Setup Summary window, is where you specify design variables, design objectives, objective constraints and penalty functions, exit criteria, and a method to optimize your system under evaluation (SUE). This interface is described in related topics.

The assumption in using the Optimization Setup Summary window is that you are working in a non-linear way to set up your criteria for optimization, moving among the tabs as you decide which values to change or set.

Preparing for Optimization
 Ultimately, before optimization can begin in ASAP, there are two tasks you must accomplish.
  1. Define at least one design variable and confirm that it is enabled.
  2. Define at least one design objective and confirm that it is enabled.

Optionally, you can define objective constraints and penalty functions. When a penalty function is assigned to an enabled objective constraint, ASAP applies the penalty function to the Figure of Merit during any iteration of the script in which the constraint is violated.

Confirming the Optimization Method

Depending on the number of variables you selected and the name of the INR file you are using for optimization, ASAP assigns a default for the method of optimization. Confirm that the selected optimization method and title are the one you want before selecting the Start Optimization button.

If you select one design variable, ASAP assigns the Brent's method as the default method.

If you select more than one variable, ASAP assigns Downhill Simplex as the default method, which you can change to Simulated Annealing.

As you continue to enable or disable design variables, ASAP automatically changes the optimization method to Brent's Method for exactly one variable or, for more than one variable, to whichever method was last selectedâ€”either Downhill Simplex or Simulated Annealing. If Downhill Simplex is selected, any associated fields must be completed. If Simulated Annealing is selected, any associated fields must contain valid data.

ASAP enters the name of the optimized INR file as the Optimization Session Title on the Optimization Method tab. You can change the title.

Validating Data

As you enter data, ASAP attempts to validate it, and either enables the Start Optimization button or issues informational messages about invalid data.

Starting Optimization

After validating selections and defaults for your design variables, design objectives, optimization method and title, ASAP enables the on the Optimization Method tab. When the button is enabled, you can select it to begin optimization.
