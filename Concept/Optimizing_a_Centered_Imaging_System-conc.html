Optimizing a Centered Imaging System

A lens can be optimized using a brute-force algorithm that finds either the nearest minimum or (given enough time) the global minimum, without requiring the merit function to be continuous (that is, no derivatives are used).

This technique also allows constraints to be handled in a straightforward and exact manner. This is made possible by an extremely fast method for determining the RMS spot size at any field location (that is, it does not have to do iterative ray-aiming to hit the internal stop surface of a wide angle lens). On a typical system, over a million-merit function evaluations per minute can be achieved, independent of the number of fields.

ASAP also allows programming of a range of variables and user-defined merit functions for custom optimization of imaging or non-imaging systems. This can be accomplished either by the grid search method, or automatically using Singular Value Decomposition (SVD) techniques. In this way, it is possible to alter an optical design, and analyze the results in the context of the larger system.

The local or global optimization feature in ASAP is not intended to replace a full-blown lens-design program. The feature is currently limited to unvignetted, centered, focal systems (the vast majority of manufactured units), and does not have any built-in multi-configuration or tolerancing capability. However, these and other advanced capabilities can be easily emulated with some additional macro-language programming (for example, $ITER). At the very least, this specific optimization feature can be used to quickly find (possibly unique) designs that may then be tweaked (if necessary) elsewhere.
