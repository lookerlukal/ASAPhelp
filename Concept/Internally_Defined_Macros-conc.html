Internally Defined Macros

The internally defined macros are contained within the parser (and therefore are always available). These macros give the program looping, input/output redirection, conditional (IF) processing, branching, and other advanced capabilities. Internally defined macros may be used freely in both user-defined macros and general input commands and may appear more than once in a given record.

Some suggestions for maximizing the execution speed of user-defined macro commands include:


  • Do not put too many definitions in one macro library file. If possible, break up a large library file into a number of smaller files, and then use the $IO LIBRARY command to switch between them.
  • Order your macro definitions within the library file from smallest to largest, or most used to least used.
  • Put several commands on each macro definition record in order to keep the the total number of records in the macro down to a minimum.
  • If possible, use registers to pass variables instead of the macro arguments. A macro that is called with no arguments is executed faster than one with arguments.

Several useful macros exist that are internally defined by the program. Do not define a macro with the same name as any of these internal macros. Also, these macros differ from the user-defined ones in that more than one can be placed on a single input record.
