CADEXPORT (ASAP Command)

Writes out CAD-compatible system geometry file.

Syntax 

CADEXPORT [ IGS [ name          ] ]
            DXF        [ block ]
            VCR

OptionDescription
IGSwrites out a fixed IGES file (the default)
DXFwrites out an AutoCAD DXF file
VCRwrites out a vector rewind VCR file
namenames the file
blockholds the DXF file standards in the named file

Remarks


  1. This command was formerly named "CAD"; however, CAD is an accepted abbreviation for CADEXPORT.
  2. Writes a 3D CAD surface representation of the currently considered objects, or the 3D graphics in the current VCR file.
  3. CADEXPORT supports the fixed IGES IGS (the default) and AutoCAD 12 DXF file standards (including the option to put everything into a named "block"). The file created is name followed by one of these extensions.
  4. Trimming information (LIMITS, BOUNDS) is translated as of ASAP 2006 V1R1.
  5. The current limitations on surface export are:
    • Advanced surface effects (REDEFINE SURF/NORM, DEFORM) are not translated.
    • IGS usually requires that the system UNITS must have already been set.
    • All base surfaces in DXF are, by necessity, approximated by facets.
    • LENS, connected CURVE/EDGE, and SAMPLED base surfaces are exported exactly to IGES. Since no CAD standard supports general algebraic surfaces, all other SURFACES are approximated by cubic spline patches or revolved cubic spline curves when possible.
    • An often better translation to AutoCAD can be done by post-processing the VCR file created by PLOT FACETS or VUFACETS with the VECT2DXF.EXE command prompt utility.
